# Contributing to Mail-To

Thank you for your interest in the Mail-To Open Source Digital Trust Delivery System! We welcome contributions from the community to help us build the standard for verifiable digital delivery.

## ü§ù How to Contribute

### 1. Reporting Bugs
*   **Security Vulnerabilities**: Please **DO NOT** file public issues. See [SECURITY.md](SECURITY.md).
*   **Functional Bugs**: Open a GitHub Issue with:
    *   Browser/OS version.
    *   Steps to reproduce.
    *   Console logs (if applicable).

### 2. Feature Requests
*   We prioritize features that enhance **Privacy**, **Verifiability**, and **Simplicity**.
*   Please discuss major architectural changes in an Issue before submitting a PR.

### 3. Development Setup

1.  **Fork & Clone**:
    ```bash
    git clone https://github.com/<YOUR_USERNAME>/mail-to.git
    cd mail-to/opensource_release
    ```

2.  **Install Dependencies**:
    ```bash
    npm install
    ```

3.  **Local Development**:
    ```bash
    npm run dev
    ```

### 4. Pull Request Process
1.  Create a feature branch (`git checkout -b feature/amazing-feature`).
2.  Commit your changes (atomic commits preferred).
3.  Push to your fork.
4.  Open a Pull Request against the `main` branch.
